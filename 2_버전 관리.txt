git은 저장소의 내용이 변경될 때마다 간단한 메모(커밋 메세지)와
수정된 내용을 '스냅샷'으로 찍어서 버전을 관리

그 후 이전 버전으로 되돌리는 방법도 제공 


git 저장소 구조)
1. 작업 트리 : 파일 수정 및 저장을 하는 폴더를 의미
2. 스테이지 : 버전(커밋)으로 생성될 파일들이 대기하는 곳
3. 저장소 : 스테이지에 대기하던 파일이 버전으로 만들어져 저장되는 곳


주요 명령어) ※ vscode를 쓰면 굳이 외울 필요없음
1. git status : 저장소 상태를 확인 (추가/수정/삭제 변화 추적)
2. git add : 변경된 파일을 스테이지에 올린다
3. git commit -m "메세지" : 스테이징된 파일을 저장소에 등록하며 버전 생성
4. git log : 버전 목록을 확인


git log 내용)
1. 커밋 해시 : 버전을 구분할 고유 번호 (외우는거 아니다)
2. author : 버전을 생성한 사람
3. date : 버전을 생성한 날짜
4. 메세지 : 버전 생성 메세지
.5 HEAD -> master : 가장 최신 버전을 나타냄
